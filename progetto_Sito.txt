Descrizione Generale
Il sito web "Speranza Felina" è stato sviluppato per facilitare le adozioni e il volontariato del gattile. È stato realizzato usando PHP per il backend e Vanilla JS insieme a React per il frontend.

Scelte Progettuali e Architetturali

1. Comunicazione tra React, Vanilla JS e PHP (Pagina Gatti)
La pagina gatti.php è il cuore dell'applicazione e dimostra l'integrazione di tecnologie diverse:
- Il modulo React (GattiApp.jsx) recupera in modo asincrono i dati dei felini tramite l'endpoint API 'api/gatti.php'. React si occupa esclusivamente del rendering dinamico delle "card", del filtraggio (per nome/descrizione) e dell'ordinamento (per età, colore o data).
- Quando un utente autenticato clicca su una card, React aggiorna il suo stato interno e lancia un evento CustomEvent('catsSelected') sull'oggetto document.
- Questo approccio disaccoppiato è stato scelto perché permette al form di prenotazione visita (gestito interamente in Vanilla JS) di rimanere indipendente dal ciclo di vita di React. Il codice Vanilla JS si iscrive all'evento 'catsSelected', riceve l'array dei gatti selezionati e aggiorna dinamicamente l'interfaccia utente (UI) e un input hidden.
- Alla sottomissione, i dati passano al backend PHP. Questa netta separazione assicura modularità: React gestisce la view complessa (lista filtrabile), mentre Vanilla JS gestisce i form classici.

2. Separazione dei Privilegi DB
L'accesso al database è stato rigorosamente suddiviso in base al principio del privilegio minimo (Least Privilege):
- Utente 'lecture' (solo SELECT): Usato per la visualizzazione della Home (ultimi arrivi), il login, e per le API di lettura dei gatti per il componente React.
- Utente 'registrator' (solo INSERT sulla tabella utenti): Usato esclusivamente dallo script di registrazione. In caso di compromissione di questa pagina, un attaccante non potrebbe leggere le password degli altri utenti.
- Utente 'modifier' (SELECT, INSERT, UPDATE): Usato per le funzionalità di inserimento di nuovi gatti da parte degli amministratori e per la registrazione dei turni di volontariato.
L'utente 'admin' o root non viene mai usato nelle connessioni applicative.

3. Form e Validazione
La validazione client-side è stata affidata interamente a Vanilla JS (come richiesto) tramite listener onsubmit. Ciò offre un feedback immediato all'utente senza ricaricare la pagina. Ad esempio, per la registrazione, sono state usate espressioni regolari per verificare la complessità della password. Nel volontariato, Vanilla JS interroga asincronamente il server (via fetch API) e disabilita a priori le fasce piene, migliorando l'UX.

4. Sicurezza
Si è fatto uso esteso di query preparate (PDO) per prevenire SQL injection in tutti i form e gli endpoint API. Le sessioni PHP sono utilizzate per gestire l'autenticazione, e per la funzionalità "Ricordami" viene impostato un cookie per 72 ore (che salva unicamente lo username per riempire il form, mantenendo però la sicurezza del login basato su password al momento del vero submit).

5. Formattazione e Accessibilità
Si è utilizzato CSS Vanilla coerente (style.css), evitando framework esterni, strutturando il layout tramite Flexbox/CSS Grid per una presentazione "responsive". Header e Footer sono dinamicamente inclusi in ogni pagina per mantenere la coerenza visiva. Le card sono disegnate in modo moderno, con hover effects visivi per migliorare l'usabilità. Le icone sono placeholder leggeri inseriti tramite inline SVG. Tutti i tag HTML5 sono usati semanticamente.
